Aruba Clearpass Policy Manager

The Aruba Clearpass Policy Manager app requires you to create an oAuth api user for SOAR to access actions.
These can be created using a guide available on the aruba developer site or here
You will need both the URL of the Publisher (base_url) as well as the client_id and client_secret.
Operating mode should be "Clearpass REST API"
Grant Type should be "Client Credentials"
If the client_secret is not generated on the screen during the creation process or you are using an existing user. You can regenerate the client_secret.
If you want to use terminate session you will require an api user with the privileges to access these functions otherwise Operator Profile "Read Only Admin API" will be sufficient
